  • Pi Hole (DNS):

    In case you don't know PiHole is a project that was originally intended to be run on a rasberry pi. It is just a DNS service that will block ads for you. The way it does that is by having a block list of advertising servers, if a DNS request is made to resolve one of these domain names it will return a null IP address. I don't own a rasberry pi so the way I deployed it was using a docker container on an ubuntu server. I used ubuntu because it was free on aws and is widely supported so that if I get stuck I can get help very easily. Also, if you use windows then you docker has to virtualise the linux kernel on top of windows. After this I just used the Network Chuck tutorial and "borrowed" the script. Btw the script is a bit broken in the network chuck video

  • Caching Server:

    If you don't know a caching server is a server that is in the way of the actual download server. This means that if you want to download a file that is installed on the download server then the caching server will check if the file is stored in it's cache. If the file is found in the cache then it will be sent across the local network that is normally faster than your internet and saves you bandwidth. In my deployment I was going to leave it running overnight on my laptop and then in the morining I would go on my computer and download the games in the cache. I used a program that I found on github to download games to the cache. I no longer use this because the server was not working and giving me really slow download speeds that meant it was a similar speed to just download the file normally.
